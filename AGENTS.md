### Key Fixes This Session
1. **Protocol CPI conditional**: `protocol.rs` functions no longer return `ProtocolCpiError` when remaining accounts are absent. They skip CPI and just update bookkeeping (`protocol_routed_underlying`). `before_value_query` returns `Ok(())` when no remaining accounts.
2. **adapter_cpi.rs order**: `cpi_deposit` had indices 4 (`vault_token_account`) and 5 (`vault_authority`) swapped. Kamino `Deposit` accounts struct expects: user, vault_state, user_position, user_token_account, **vault_authority**, **vault_token_account**, token_program, system_program.
3. **Maple test mint consistency**: `before` hook mint now passed through to `runAdapterDepositWithdrawFlow` on all networks.
4. **Vault authority seed mismatch**: Each adapter uses a custom `VAULT_AUTHORITY_SEED` (e.g., `b"kamino_vault_authority"`) but the dispatcher's `adapter_validation::expected_vault_authority_pda` used the trait's hardcoded `b"vault_authority"`. Fixed by adding `vault_authority_seed` to `AdapterEntry` state and `propose_adapter` instruction, and updating dispatcher validation to read it from the registry entry.
5. **Drift slippage-before-cooldown**: Binary was stale (built before source fix). Fixed: generate new program keypair, `anchor keys sync`, rebuild, deploy under new ID `2zMNZcFzAx9bFNchTWDqiJGt5H3bCDgo8PW1TTskwcLJ`.
6. **Stale program-level keypair files**: `programs/*/target/deploy/*-keypair.json` files (generated by previous builds) conflicted with Anchor's keypair check. Required deleting them and copying the correct keypair from `target/deploy/` before each build.
