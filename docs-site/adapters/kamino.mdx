## Why Kamino is the golden reference

Start here when building your own adapter. `programs/adapter-kamino/` is the smallest complete implementation of:

- `deposit` / `withdraw` / `current_value`
- `define_adapter_position!` for per-user PDAs
- Vault authority PDA signing outbound transfers
- Token mint constraints on deposit and withdraw

## Instruction summary

| Instruction | Input | Effect |
|---|---|---|
| `deposit` | Underlying amount | Mint receipt shares; transfer USDC user ‚Üí vault |
| `withdraw` | Receipt shares | Burn shares; transfer pro-rata USDC vault ‚Üí user |
| `current_value` | ‚Äî | Emit user's underlying balance (not global share price) |

## Key accounts

| Account | Description |
|---|---|
| `vault_state` | `KaminoVaultState` PDA ‚Äî `total_underlying`, `total_shares` |
| `user_position` | Per-user `AdapterPosition` ‚Äî `receipt_token_balance` |
| `vault_token_account` | SPL token account owned by vault authority PDA |
| `vault_authority` | PDA signer for outbound vault transfers |

## PDA seeds

```
vault_state:     ["kamino_vault_state"]
vault_authority: ["kamino_vault_authority"]
user_position:   ["adapter_position", user_pubkey]
```

## Production path

A production Kamino adapter would CPI `deposit_reserve_liquidity` and `redeem_reserve_collateral` on the K-Lend program, passing reserve and market accounts via `remaining_accounts`. Fork tests in this repo clone the Kamino program ID to validate deployment plumbing ‚Äî not live position parity.
