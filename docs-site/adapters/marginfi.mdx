## Overview

MarginFi v2 uses a Just-In-Time (JIT) risk engine over bank accounts and oracles. This reference adapter captures the **share receipt model** without integrating MarginFi's account graph.

## Instruction summary

| Instruction | Input | Effect |
|---|---|---|
| `deposit` | Underlying amount | Credit receipt shares; SPL transfer to vault |
| `withdraw` | Receipt shares | Burn shares; return pro-rata underlying |
| `current_value` | — | Per-user underlying value from `receipt_token_balance` |

## PDA seeds

```
vault_state:     ["marginfi_vault_state"]

```

## Production path

Production CPI would call `lending_account_deposit` / `lending_account_withdraw` with `remaining_accounts` for banks, oracles, and the user's MarginFi account. The JIT risk engine evaluates collateralization at transaction time — no separate refresh instruction.
