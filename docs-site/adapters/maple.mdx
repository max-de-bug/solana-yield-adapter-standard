## Overview

Maple's syrupUSDC pools use a share-based NAV that accrues as loans perform. Because Maple has limited Solana deployment, this adapter:

1. Implements the full three-instruction standard
2. Accrues yield on `total_underlying` over elapsed slots (demonstration only)
3. Serves as a template for cross-chain protocols launching on Solana

## Instruction summary

| Instruction | Input | Effect |
|---|---|---|
| `deposit` | Underlying amount | Mint shares at current NAV |
| `withdraw` | Receipt shares | Burn shares at current NAV |
| `current_value` | — | User underlying value after simulated accrual |

## When to use this pattern

Teams building adapters for protocols that:

- Are deploying from EVM to Solana
- Have CPI under development but need a working standard-compliant program today
- Want to demonstrate share-price appreciation without live protocol state
