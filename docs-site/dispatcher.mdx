The **Yield Dispatcher** is the single entry point for integrators. Users and dApps call the dispatcher; it validates registry approval, checks the circuit breaker, then CPIs into the target adapter. All position accounting lives in the adapter — the dispatcher is a thin pass-through.

<Info>
**Devnet program ID:** `8u4YFQiTCR5n5dijVoinXyZ962ngVmFuWKELDUjVCqAR`
</Info>

## Design Principle

The dispatcher is intentionally **minimal**: it validates, routes, and emits events. It does **not** maintain internal position ledgers or duplicate adapter state. This avoids the "parallel accounting" problem where a dispatcher-level `UserPosition` can diverge from the adapter-level `AdapterPosition`.

The flow for every instruction is:

1. **Check circuit breaker** — `is_paused` gate
2. **Validate registry** — adapter must have `status == Approved`
3. **CPI into adapter** — forward accounts and arguments as-is
4. **Increment counter** — `total_deposits` / `total_withdrawals` (monitoring only)
5. **Emit event** — indexer surface

## Instructions

### `initialize`

Creates the global `DispatcherState` PDA. Called once at deployment.

```typescript
await program.methods.initialize().accounts({
  authority: wallet.publicKey,
  dispatcherState: dispatcherStatePda,
  registryProgram: registryProgramId,
  systemProgram: SystemProgram.programId,
}).rpc();
```

### `deposit`

Routes a deposit to an approved adapter via CPI.

<ParamField path="amount" type="u64" required>
  Underlying token amount to deposit through the adapter.
</ParamField>

<ParamField path="min_shares_out" type="u64" required>
  Minimum receipt shares to accept. Transaction reverts with `SlippageExceeded` if the calculated shares are below this threshold. Pass `0` to disable protection.
</ParamField>

**Preconditions:**

- Dispatcher not paused (`is_paused == false`)
- Adapter `Approved` in registry
- Valid mint on user token account
- Adapter vault accounts pass PDA validation (`adapter_validation` module)

### `withdraw`

Routes a withdrawal. The `shares` argument is forwarded to the adapter as **receipt shares to burn**.

<ParamField path="shares" type="u64" required>
  Receipt shares to burn.
</ParamField>

<ParamField path="min_underlying_out" type="u64" required>
  Minimum underlying tokens to receive. Transaction reverts with `SlippageExceeded` if the calculated amount is below this threshold. Pass `0` to disable protection.
</ParamField>

### `current_value`

CPIs the adapter's read-only `current_value` instruction. Returns per-user underlying value via `CurrentValueEvent`.

### `toggle_pause`

**Authority only.** Flips the dispatcher's emergency pause flag. When paused (`is_paused == true`), both `deposit` and `withdraw` routing are blocked. This is a global circuit breaker — individual adapters retain their own `toggle_status` for per-vault granularity.

This two-layer safety model matches the pattern used by Kamino (`emergency_mode`) and Marginfi.

## State accounts

### `DispatcherState`

| Field | Type | Description |
|---|---|---|
| `authority` | `Pubkey` | Governance authority (multisig recommended) |
| `registry_program_id` | `Pubkey` | Trusted registry program |
| `total_deposits` | `u64` | Cumulative deposits routed (monitoring only) |
| `total_withdrawals` | `u64` | Cumulative withdrawals routed (monitoring only) |
| `is_paused` | `bool` | Emergency pause flag |
| `bump` | `u8` | PDA bump seed |

The dispatcher has **no user-level position accounts**. All position tracking is delegated to the adapter's `AdapterPosition` PDAs (see [Adapter Standard](/adapter-standard#44-pda-reference-table)).

## Events

| Event | Purpose |
|---|---|
| `DispatchDepositEvent` | Indexer signal: deposit routed to adapter |
| `DispatchWithdrawEvent` | Indexer signal: withdraw routed to adapter |

Adapter-level [`DepositEvent`](/reference/events) / [`WithdrawEvent`](/reference/events) are emitted by the adapter CPI and contain the exact number of shares minted or underlying returned.

## Integration pattern

```typescript
// Deposit USDC through dispatcher into Kamino adapter
await dispatcherProgram.methods
  .deposit(new BN(1_000_000), new BN(0)) // amount, min_shares_out
  .accounts({
    user: wallet.publicKey,
    dispatcherState,
    registryProgram: REGISTRY_ID,
    adapterProgram: KAMINO_ADAPTER_ID,
    adapterEntry: kaminoEntryPda,
    // ... adapter vault accounts, user token accounts,
    // adapterUserPosition (tracked by adapter, not dispatcher)
  })
  .rpc();
```
