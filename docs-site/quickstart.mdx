Get from zero to a passing test suite with the commands below. For toolchain details, see the [GitHub README](https://github.com/max-de-bug/solana-yield-adapter-standard#prerequisites).

## Prerequisites

<AccordionGroup>
  <Accordion title="Rust 1.75+">
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    rustup default stable
    ```
  </Accordion>
  <Accordion title="Solana CLI 2.2.20">
    ```bash
    sh -c "$(curl -sSfL https://release.solana.com/v2.2.20/install)"
    solana --version
    ```
  </Accordion>
  <Accordion title="Anchor CLI 1.0.1">
    ```bash
    cargo install --git https://github.com/coral-xyz/anchor avm --force
    avm install 1.0.1 && avm use 1.0.1
  agave-install init 2.2.20
    ```
  </Accordion>
  <Accordion title="Node.js 18+">
    Required for TypeScript integration tests and npm scripts.
  </Accordion>
</AccordionGroup>

## Clone and build

```bash
git clone https://github.com/max-de-bug/solana-yield-adapter-standard.git
cd solana-yield-adapter-standard
npm install
npm run build
```

<Tip>
`npm run build` runs `scripts/build-sbf.sh` (all seven programs) then `scripts/build-idls.sh` (IDL artifacts under `target/idl/`).
</Tip>

## Run tests

<Tabs>
  <Tab title="Local (recommended)">
    ```bash
    npm test
    ```

    Runs **17 integration tests** against a local validator:

    - Registry governance lifecycle
    - Dispatcher routing and pause
    - All five adapters: deposit → `current_value` → withdraw
    - Error paths (zero amounts, insufficient balance)
  </Tab>
  <Tab title="Mainnet fork">
    ```bash
    ./scripts/run-fork-solana-test-validator.sh
    ```

    Runs **31 integration tests** on a validator with cloned mainnet programs (Kamino, MarginFi, Drift, Jupiter Perps, Maple, SPL Token) and injected fixture ATAs.

    <Warning>
    Requires network access to clone program state. First run may take several minutes.
    </Warning>
  </Tab>
</Tabs>

## Deploy to devnet

```bash
npm run deploy:devnet
# or: ./scripts/deploy-devnet.sh
```

The script generates keypairs, builds programs, deploys to devnet, and prints program IDs to paste into `Anchor.toml`.
