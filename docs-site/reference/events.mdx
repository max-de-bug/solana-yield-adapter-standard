---
title: Events
description: Standardized on-chain events for indexing and monitoring
---

All adapters emit the same three events. The dispatcher emits additional routing events. Index either adapter-level events (position detail) or dispatcher events (routing volume).

## Adapter events

### `DepositEvent`

Emitted on successful `deposit`.

```rust
#[event]
pub struct DepositEvent {
    pub user: Pubkey,
    pub adapter: Pubkey,
    pub amount: u64,           // Underlying deposited
    pub receipt_amount: u64,   // Shares minted
    pub timestamp: i64,
}
```

### `WithdrawEvent`

Emitted on successful `withdraw`.

```rust
#[event]
pub struct WithdrawEvent {
    pub user: Pubkey,
    pub adapter: Pubkey,
    pub amount: u64,           // Underlying returned
    pub receipt_burned: u64,   // Shares burned
    pub timestamp: i64,
}
```

### `CurrentValueEvent`

Emitted on `current_value` query.

```rust
#[event]
pub struct CurrentValueEvent {
    pub user: Pubkey,
    pub adapter: Pubkey,
    pub value: u64,            // User position in UNDERLYING units
    pub timestamp: i64,
}
```

<Warning>
`value` is the caller's position in underlying token units (`user_position_underlying_value`), **not** the global share price.
</Warning>

## Dispatcher events

### `DispatchDepositEvent`

```rust
pub struct DispatchDepositEvent {
    pub user: Pubkey,
    pub adapter_program_id: Pubkey,
    pub amount: u64,
    pub timestamp: i64,
}
```

### `DispatchWithdrawEvent`

```rust
pub struct DispatchWithdrawEvent {
    pub user: Pubkey,
    pub adapter_program_id: Pubkey,
    pub amount: u64,
    pub timestamp: i64,
}
```

## Listening in TypeScript

```typescript
program.addEventListener("DepositEvent", (event, slot) => {
  console.log(`Deposit ${event.amount} → ${event.receipt_amount} shares`);
  console.log(`User: ${event.user.toBase58()}, slot: ${slot}`);
});

program.addEventListener("CurrentValueEvent", (event) => {
  console.log(`Position value: ${event.value} underlying`);
});
```

## Indexer notes

Events appear in transaction logs as Anchor-emitted data. Any Solana indexer (Helius, Triton, custom Geyser plugin) can decode them using the program IDL from `target/idl/`.

Generate fresh IDLs: `npm run build:idls`
