The test suite validates registry governance, dispatcher routing, and all six adapters (5 reference + 1 template) end-to-end. Run locally before every PR.

## Test matrix

| Command | Tests | Environment | Duration |
|---|---|---|---|---|
| `npm test` | 32 (26 passing, 6 pre-existing slippage failures on localnet) | Local validator | ~2 min |
| `bash scripts/run-fork-surfpool.sh` | 112 executable (124 registered, 12 skipped¹) | Surfpool (JIT account fetching) | ~5–15 min |

¹ 96 static `it()` declarations; shared helpers (`runConformance`, `addSlippageTests`) dynamically register more at runtime, bringing the fork total to 124. 11 Drift CPI tests and 1 Maple vault-lifecycle check are skipped — see [test count breakdown](#test-count-breakdown) below.

<Tip>
`npm test` uses `anchor test --validator legacy --skip-build`. Run `npm run build` first if programs changed.
</Tip>

## Mainnet-fork tests (Surfpool)

Uses [Surfpool](https://surfpool.run) for JIT account fetching — no manual `--clone` flags or fixture ATAs needed:

```bash
# Prerequisites
curl -sL https://run.surfpool.run/ | bash

# One-time setup — create .env (gitignored) with your RPC key
cp .env.example .env
# Edit .env: MAINNET_RPC_URL=https://mainnet.helius-rpc.com/?api-key=YOUR_KEY

# Run (auto-loads MAINNET_RPC_URL from .env)
bash scripts/run-fork-surfpool.sh
```

The script (`scripts/run-fork-surfpool.sh`):

1. Builds all programs and IDLs
2. Starts a Surfpool validator (auto-fetches mainnet accounts on demand)
3. Deploys all programs to the fork
4. Runs `MAINNET_FORK=1 anchor test --skip-local-validator --skip-build`

All **112 tests pass** on fork (the 6 slippage failures seen on localnet are resolved on fork because JIT-fetched USDC ATAs provide the correct underlying mint). The full suite covers all adapters (Kamino, MarginFi, Jupiter, Maple, Drift, template), dispatcher routing with CPI-based `current_value`, complete registry lifecycle (including re-approval from `Revoked`), and Drift two-phase withdrawal with stale-ticket cleanup.

### Test count breakdown

The static `it()` count across all test files is **96**, but shared helpers register additional tests per adapter at runtime, bringing the fork total to **124 registered**. Of those, **12 are skipped** on fork:

| Component | Registered | Skipped | Executable |
|---|---|---|---|
| Jupiter adapter | 18 | 0 | 18 ✅ |
| Kamino adapter | 18 | 0 | 18 ✅ |
| Marginfi adapter | 18 | 0 | 18 ✅ |
| Template adapter | 18 | 0 | 18 ✅ |
| Maple adapter | 17 | 1 | 16 ✅ |
| Drift adapter | 18 | 11 | 7 ✅ |
| Dispatcher | 11 | 0 | 11 ✅ |
| Registry | 13 | 0 | 13 ✅ |
| **Total** | **124** | **12** | **112 ✅** |

- **Drift (11 skips):** 5 `it.skip` (CPI instructions disabled upstream — Drift v2 program has all instruction handlers commented out) + 6 `this.skip` in shared conformance tests (safe to bypass when protocol CPI is non-functional). See `Docs/troubleshooting/drift-fork-issues.md`.
- **Maple (1 skip):** Vault-lifecycle conformance check uses a non-standard status model.
- Once Drift re-enables its program, these 11 tests will pass without code changes.

### Surfpool-specific features

- **State persistence:** Surfpool preserves accounts across restarts. The registry's `force_transfer_governance` instruction (admin-gated escape hatch) resets stale governance from prior runs.
- **Latency:** Surfpool's 400ms slot time requires `sleep()` delays between rapid RPC calls. The test suite uses 3000ms delays where blockhash reuse might occur.
- **Drift stale tickets:** `clearPendingTicket` cancels stale withdrawal tickets from prior Surfpool sessions before creating new ones.

<Warning>
Fork tests prove **deployment plumbing and USDC flows** against cloned bytecode. They do **not** prove live mainnet position parity inside those protocols. See [Reference vs Production](/guides/reference-implementation).
</Warning>

### Folder consolidation

Program sources live directly in `programs/` (no symlinks). The `adapters/` directory was removed — workspace members in `Cargo.toml` reference `programs/` paths. The `adapter-template` at `programs/adapter-template/` provides a ready-to-copy scaffold.

## Writing new tests

Minimum cases for any adapter:

```typescript
it("deposit → current_value → withdraw", async () => {
  await deposit(1_000_000);
  const value = await currentValue();
  expect(value).to.be.gt(0);
  await withdraw(shares);
});

it("rejects zero deposit", async () => {
  await expect(deposit(0)).to.be.rejected;
});
```
