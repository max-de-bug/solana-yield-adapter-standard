## Devnet (quick)

```bash
npm run deploy:devnet
```

The script (`scripts/deploy-devnet.sh`):

1. Builds all seven programs via `scripts/build-sbf.sh` + `scripts/build-idls.sh`
2. Deploys registry, then dispatcher, then all 5 adapters to devnet
3. Prints program IDs for `Anchor.toml`

<Info>
Ensure your Solana CLI points to devnet and your wallet has ~2.6 SOL for a full deploy:
`solana config set --url devnet && solana balance`
</Info>

## Post-deploy initialization

<Steps>
  <Step title="Initialize registry">
    ```typescript
    await registryProgram.methods.initialize().accounts({
      authority: wallet.publicKey,
      registryState: registryStatePda,
      systemProgram: SystemProgram.programId,
    }).rpc();
    ```
  </Step>
  <Step title="Initialize dispatcher">
    ```typescript
    await dispatcherProgram.methods.initialize().accounts({
      authority: wallet.publicKey,
      dispatcherState: dispatcherStatePda,
      registryProgram: REGISTRY_ID,
      systemProgram: SystemProgram.programId,
    }).rpc();
    ```
  </Step>
  <Step title="Register adapters">
    For each adapter: `proposeAdapter` then governance `approveAdapter`.
  </Step>
  <Step title="Transfer governance">
    Move registry and dispatcher authority to a Squads multisig.
  </Step>
</Steps>

Full registry flow: [Registry](/registry).

## Mainnet

<Warning>
**Pre-mainnet checklist:**

- Independent security audit of all programs
- Hardware wallet or multisig as deployer and governance
- Monitoring for dispatcher and adapter events
- Governance transfer before public launch
</Warning>

```bash
solana config set --url mainnet-beta
npm run build
anchor deploy --provider.cluster mainnet-beta
```

Update `[programs.mainnet]` in `Anchor.toml` with deployed IDs and regenerate client code from fresh IDLs (`npm run build:idls`).
