## Overview

Alongside this Anchor reference implementation, we maintain an equivalent port built with the [**Quasar**](https://github.com/blueshift-gg/quasar) framework. Quasar is a `#![no_std]` Solana framework with zero-copy serialization, designed as a lighter alternative to Anchor.

The Quasar port lives in the [`syas-quasar`](https://github.com/max-de-bug/syas-quasar) repository and implements the exact same interfaces, accounts, and tests.

## Head-to-Head Results

Measured on identical hardware (Ubuntu 22.04, Ryzen 9, 32 GB RAM):

| Metric | Anchor | Quasar | Δ |
|--------|--------|--------|---|
| **Build time** (8 programs) | 25.6s | 20.2s | **21% faster** |
| **Total binary size** | 2.0 MB | 0.3 MB | **86% smaller** |
| **Test execution** | validator RPC (~3s/adapter) | QuasarSVM in-process (~ms) | **~100x faster** |
| **Runtime dependency** | Borsh + std | `#![no_std]` | lighter |


## Key Differences

| Area | Anchor | Quasar |
|------|--------|--------|
| Language features | `std` + Borsh + heap alloc | `#![no_std]`, zero-copy pointer casts |
| Instruction dispatch | SHA256 discriminators | Explicit `#[instruction(discriminator = N)]` |
| Account serialization | Borsh `AccountDeserialize` | `set_inner()` + `#[seeds()]` |
| Test framework | `anchor test` + mocha/TS | QuasarSVM (Rust) + TS clients |
| CPI routing | `CpiContext` with `invoke_signed` | `invoke_signed` with `Seed` helpers |
